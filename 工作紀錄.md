# KMTG 自由桌面版 — 工作紀錄

> 本檔記錄簡化版的設計決策與進度。完成的項目打勾，改變想法時直接修改本檔。

## 目標

把原本「LLM 當裁判＋AI 對手」的架構，簡化成**數位桌面**：
提供牌桌和道具，規則計算全部由玩家自己進行，程式不干涉、不裁判。

保留三樣東西：
1. `decks/` 已設定好的牌組 MD 檔
2. 製作卡牌的機制（輸入卡名 → Scryfall 抓卡圖與規則 → 存入本地資料夾）
3. 戰場顯示系統（改造成可互動桌面 `tabletop.html`）

## 設計決策

| 決策 | 內容 | 理由 |
|------|------|------|
| 雙方手牌攤開 | 兩位玩家共用同一畫面，手牌全部面朝上 | 目前先這樣做；日後再研究兩人分別用不同畫面對戰 |
| 不做 undo | 拖錯就自己拖回去 | 核心是玩家拖曳操作，過多自動化會讓玩家在自由和功能之間混淆 |
| 不做規則引擎 | 不驗證法力、不自動結算傷害、不強制階段 | 桌面只是道具；傷害與法力由玩家心算 |
| Aura 附著＝自由疊放 | 不做程式上的「附著」，直接把 Rancor 疊在生物卡上 | 符合實體桌面直覺，避免自動化混淆 |
| Token 離開戰場即消失 | 從戰場拖到其他區域時直接刪除 | 符合 MTG 規則（token 離場即消失） |
| 卡圖本地化 | 卡圖下載到 `decks/cards/` 資料夾，離線可玩；缺圖時 fallback 到 Scryfall 即時抓 | 不依賴網路、不重複打 API |
| 卡圖跟牌組放一起 | `decks/cards/` 而不是根目錄 `cards/` | 管理方便：牌組相關的東西（MD、卡表、卡圖）集中在同一層 |

## 桌面功能清單（道具）

每位玩家一套：
- **牌庫**：點擊抽 1 張；按鈕：洗牌、檢視/搜尋（攤開全部→拖出→自動提示洗牌）、看頂 X 張（占卜）、磨 1（頂牌進墳場）
- **手牌區**：面朝上，可拖到任何區域
- **戰場**：自由擺放（絕對定位拖曳）；點擊橫置/解橫；一鍵「全部解橫」
- **墳場／流放區**：可攤開檢視、拖回任何區域
- **生命計數器**：±1、±5、可直接輸入
- **Token 產生器**：常用預設（松鼠 1/1、骷髏 1/1、精靈 1/1、猿猴 4/4…）＋自訂名稱/P/T
- **指示物**：+1/+1、-1/-1、通用計數器（Echo/消退等玩家自行賦義），拖到卡上顯示徽章、跟卡移動

共用道具：
- 回合計數器（純顯示，＋1 按鈕）
- 擲硬幣、D6、D20
- 開新局：選兩副牌組 → 洗牌 → 各抽 7；「重抽 7」按鈕當作自由調度
- 滑鼠懸停：放大卡圖＋顯示 oracle text（讀 `cards.json`）
- 狀態自動存 localStorage，關掉瀏覽器可續局
